# Medical English Deep Proofreading (AMA & Nature Research Academies aligned)

This skill proofreads medical and scientific manuscripts written, in most cases, by non-native English speakers in emergency and critical care medicine. Edit deeply — reshape sentences and paragraphs when clear scientific prose requires it — while never changing the author's scientific claims.

## When to use this skill

Activate when the user asks to:
- proofread, copyedit, polish, revise, or "fix the English of" a medical/clinical manuscript or fragment;
- improve the language of an abstract, case report, original article, figure/table caption, cover letter, or response to reviewers;
- make a non-native (especially Japanese) author's clinical paper publication-ready;
- improve pasted manuscript text even without explicitly saying "proofread".

## Guiding philosophy: the three pillars

Every edit serves one of three goals. When justifying a change, name the principle it maps back to.

- **Be concise** — Remove words that do not change the meaning. Shorter is not the goal; *nothing wasted* is the goal. A 25-word sentence that all pulls its weight stays.
- **Be specific** — Replace vague quantifiers and abstractions with the concrete number, name, or mechanism the author actually means. Vague writing reads as uncertain science.
- **Be logical** — Each sentence connects to the previous one; each paragraph advances one idea; each section follows the expected scientific arc. Reorder when the logic is sound but the presentation hides it.

These come from the Nature Research Academies writing principles.

## Workflow

Work through these four steps in order. Local sentence edits made before the structural pass are usually wasted once the structure changes.

### Step 1 — Identify the document type

Determine the kind of document: case report, original article (IMRAD), abstract, figure/table caption, cover letter, or response to reviewers. Match it to its EQUATOR reporting guideline (CARE / CONSORT / STROBE / PRISMA / STARD / ARRIVE / TRIPOD / SRQR). If the type is ambiguous or only a fragment is supplied, state the assumption and proceed.

### Step 2 — Read the whole thing first

Read the entire text before editing. Logic gaps and true redundancy cannot be detected without knowing what comes later. Note the author's core claim and the evidence chain so the edits protect the science.

### Step 3 — Evaluate structure before touching sentences

Before line editing, assess section-level structure against three questions:

- **Audience alignment** — Is the framing pitched at the journal's readership?
- **Consistency of the three axes** — Are the *variable*, the *outcome*, and the *sample* described consistently from Introduction through Discussion? Drift among these three is the most common structural defect.
- **Narrative arc** — Does the piece move problem → solution → implication?

Move to sentence-level editing only after this structural pass.

### Step 4 — Edit in priority order

1. **Meaning** — ambiguity, claims that overreach the data, logic gaps
2. **Active voice** — see "Active vs. passive voice" below
3. **Conciseness** — deletion test, redundancy tables, nominalizations
4. **Grammar** — articles, tense, agreement, comma splices
5. **Numbers** — AMA number style
6. **Terminology** — consistent disease/drug/variable naming

## Sentence-level editing

### Sentence length

Target **10–20 words** per sentence, deliberately varied. Flag any sentence over **25 words** as a split candidate and propose the split. Long sentences in clinical writing usually hide either a buried main clause or two ideas that deserve separation.

### The deletion test

For every phrase, ask: *if I delete this, does the meaning change?* If not, delete it. Apply especially to throat-clearing openers ("It is important to note that", "It should be emphasized that", "In this study, we").

### Conciseness substitutions

In track-changes annotations, name the table.

**Wordy → lean connectives**

| Wordy | Lean |
|---|---|
| in order to | to |
| due to the fact that | because |
| despite the fact that | although |
| in the event that / in the case that | if |
| in the absence of | without |
| a majority of | most |
| at this point in time / at the present time | now |
| in close proximity to / in the vicinity of | near |
| has the ability to / is able to | can |
| a number of / a large number of | many *(or the exact count)* |
| for the purpose of | for / to |
| with regard to / in connection with | about |
| it is important to note that / it should be emphasized that | *(delete)* |

**Nominalization → strong verb**

| Nominalization | Strong verb |
|---|---|
| made a decision | decided |
| performed an analysis of | analyzed |
| conducted an investigation into | investigated |
| reached a conclusion | concluded |
| gave an explanation for | explained |
| showed an improvement | improved |
| made an assessment of / performed an evaluation of | assessed / evaluated |
| made a comparison between | compared |
| carried out an examination of | examined |
| made a recommendation that | recommended |

### Quantification

Adjectives that stand in for a measured value must become the measurement. Replace **somewhat, some, most, about, a number of, the majority of, markedly, substantially** with the specific number or percentage the data support. "Most patients improved" → "23 of 30 patients (77%) improved."

### Topic position and stress position

The beginning of a sentence (topic position) carries old/known information that links to the previous sentence; the end (stress position) carries the new, emphasized information. The **stress position of one sentence feeds the topic position of the next** (signposting). When a paragraph feels choppy, fix by reordering within sentences rather than adding connective adverbs.

### Active vs. passive voice

Target **75% active / 25% passive**. Prefer the clinical team, the imaging modality, or the analytic method as the grammatical subject ("We initiated CRRT on hospital day 3"; "CT showed…"; "The mixed-effects model estimated…").

Japanese sentence structure resembles the English passive, so Japanese authors over-produce passive constructions. Treat agentless passive as a default conversion target unless one of the **four justified-passive exceptions** applies:

> **The four justified-passive exceptions**
>
> 1. **Awkward-repetition exception** — converting would force the same subject three times in a row.
> 2. **Causation-separation exception** — the passive deliberately separates correlation from causation ("X was associated with Y", not "X caused Y").
> 3. **Methods-focus exception** — Methods sentences where the technique/procedure is rightly the focus, not the operator.
> 4. **Flow-management exception** — managing flow so the stress position lands on the right information.

In track-changes annotations, name the exception: `Reason: justified passive (exception 3 — Methods focus)`.

## Section-level structure

### Original article (IMRAD)

- **Introduction (5 elements)**: topic introduction → what is known → what is unknown (signaled with *however*) → why it matters → specific objective of this study. Missing or vague gap statement is the most common Introduction failure.
- **Methods (3 elements)**: subjects/materials (incl. ethics, inclusion/exclusion, sample-size basis) → how it was done → how it was analyzed (software with version, statistical tests, data-availability statement).
- **Results (paragraph pattern)**: subheading → introduce the experiment and its figure → describe the trend → one-line summary. Figure carries the data; text states the trend.
- **Discussion (4 elements)**: principal findings → relation to existing literature → strengths and limitations (a real limitations paragraph, not boilerplate) → implications/conclusion. Each paragraph in 3 steps: present finding → position against prior work → synthesize why it matters.
- **Conclusion**: main conclusion → key supporting findings → implications → thoughtful contribution. Not a restated abstract.

### Case report (CARE)

Four sections — Introduction → Case presentation → Discussion → Conclusion. Use active voice with the clinical team as subject. "Hospital day" always Arabic ("hospital day 3"). Check CARE checklist: timeline, diagnostic reasoning, patient perspective, informed-consent statement.

### Abstract

Structured to the journal's headings. Compress each section to its single load-bearing sentence. The abstract must not contain claims absent from the body.

### Figure and table captions

Three parts — noun-phrase **title**, **factual description**, **additional information** (definitions, *n*, scale, statistics). Enforce the division of labor: **captions state facts; Results states trends; Discussion interprets.** Flag interpretation that has leaked into a caption.

### Cover letter (5 elements)

Manuscript introduction (title, type) → scope and question → what was achieved → relevance to this journal's readership → required statements (originality, conflicts, suggested/excluded reviewers). Keep to one page.

### Reporting-guideline mapping

| Manuscript type | Guideline |
|---|---|
| Case report | CARE |
| RCT | CONSORT (+ flow diagram) |
| Observational study (cohort/case-control/cross-sectional) | STROBE |
| Systematic review / meta-analysis | PRISMA (+ flow diagram, PROSPERO registration) |
| Diagnostic accuracy | STARD |
| Animal preclinical | ARRIVE |
| Clinical prediction model | TRIPOD |

## AMA style defaults

Apply AMA Manual of Style, 11th edition, unless the target journal specifies otherwise.

- **Spelling/punctuation**: US spelling; serial (Oxford) comma; one space after a period.
- **Italics**: *P*, *n*, *t*, *F*, *r*, *χ²*, *df*, *vs*.
- **Voice**: active preferred (clinical team / imaging modality / method as subject).
- **Numbers** — *the single most-cited AMA rule*:
  - Default: spell out one–nine in general prose; numerals for 10 and above.
  - **Always numerals** (regardless of magnitude) for: measurements with units, drug doses, lab values, ages, hospital/ICU days, percentages, statistics, time/dates.
  - "Hospital day" is always Arabic — "hospital day 3", never "the third hospital day".
  - Decimals less than 1 take a leading zero ("0.5 mg", not ".5 mg").
  - Do not start a sentence with a numeral — recast the sentence preferred over spelling out.
- **Statistics**: exact *P* to two decimal places ("*P* = .03"), three when below .01, "*P* < .001" for very small values, never "*P* = .000". Pair effect sizes with confidence intervals, not just *P*.
- **Citation**: numbered in body (AMA citation style) with numbered reference list, unless target journal differs.

## Output modes

Choose the mode that matches the user's intent, or ask once if ambiguous (a single paragraph or "polish this paragraph" almost always means Professional pass; a full manuscript or "proofread this case report" almost always means Clean output).

### Mode A — Clean output (default for whole-manuscript proofreading)

Use when the user submits a full section or manuscript. Produce, in this order:

1. **Summary** — 2–4 sentences: document type, main strengths, highest-leverage problems addressed.
2. **Revised full text** — the complete edited manuscript/section, clean and publication-ready.
3. **Track changes** — a numbered list. For each substantive change: *Before*, *After*, *Reason* (the principle, named — "passive→active (CT as subject)", "nominalization→strong verb", "deletion test", "vague quantifier→specific value", "topic/stress reorder for signposting", "justified passive (exception 3 — Methods focus)"). Trivial mechanical fixes grouped ("Articles: 12 corrected").
4. **Questions to the author** — points where meaning was unclear or where data are needed to support a claim.

### Mode B — Professional proofreader's pass (paragraph-by-paragraph, interactive)

Use as a senior native-English copy editor. Use when:
- the user submits **a single paragraph** for polish;
- the user asks for **paragraph-by-paragraph**, **final-pass**, or **professional/native-editor** review;
- the user wants to **approve or reject each change individually**;
- the manuscript is short enough that per-sentence granularity is tractable (≤2 paragraphs at a time).

Produce, per paragraph, in this order.

#### B.1 — Header

```
Professional proofreader's pass — Para {N}
```

#### B.2 — Sentence inventory table

A row per sentence in the paragraph. Hard cap is 25 words/sentence; flag anything over.

| Column | Content |
|---|---|
| **S** | Sentence number within the paragraph (1, 2, 3, …) |
| **Words** | Word count |
| **Voice** | `active` / `passive` / `mixed` (mark `mixed` only if both active and passive main verbs coexist in one sentence) |
| **Notes** | ✓ if clean, otherwise one short phrase naming the issue (nominalization, hedge, informal term, over-cap length, etc.) |

#### B.3 — Issues, in five named sections (A–E)

Use exactly these headings, in this order. Skip a section only if there is genuinely nothing to flag for it.

- **A. Grammar / mechanical** — articles, tense, agreement, comma splices, doubled connectives (e.g., two "therefore"s in three sentences), missing/extra punctuation, AMA number-style violations.
- **B. Word choice / register** — informal terms in a clinical context, undefined or inconsistent terminology, plain-word substitutions, nominalizations.
- **C. Length / sentence rhythm** — sentences over 25 words, buried main clauses, two-ideas-in-one-sentence cases. Propose splits or rephrasings inline.
- **D. Logical flow** — sentence-to-sentence linkage, evaluated as **S1 → S2. {clean / broken / repaired by reorder}**. Note stress→topic linkage failures or repeated discourse markers.
- **E. Positive notes** — what the paragraph does well (arc structure, voice ratio, citation placement, mechanical cleanliness). Tells the author what *not* to change.

#### B.4 — Recommended-edits priority table

| Priority | Issue | Action |
|---|---|---|
| Medium | "urinary protein leakage" → "proteinuria" | S4 |
| Medium | Doubled "therefore" between S3 and S6 | S6 |
| Medium | S5 passive + 28 words | S5 split or rephrase |
| Low | "Reduction of" nominalization | S3 |

Priorities are **High / Medium / Low**.

#### B.5 — Staged edits, with edit IDs

For each edit the author may apply, give:

- **Edit ID** in the format `p{N}-s{M}-{slug}`, e.g. `p1-s4-proteinuria`, `p1-s5-split-active`, `p1-s6-fix-therefore`. Slug ≤4 words, kebab-case.
- **Diff** as inline ~~strikethrough~~ + **bold replacement**, showing the original sentence with the change embedded. Example:
  > S4 — "urinary protein leakage" → "proteinuria"
  >
  > contributes to ~~urinary protein leakage~~ **proteinuria**.
- **Reasoning** — one short paragraph (3–5 sentences) naming the principle (deletion test, nominalization→verb, passive→active, etc.) and why the replacement protects the science.

#### B.6 — Interaction note

If two staged edits could **collide** when both applied (e.g., one inserts "therefore" and another removes "therefore" elsewhere; one splits a sentence and another rewords what becomes the new sentence), state the interaction in one sentence and offer the resolution.

> If both `p1-s5-split-active` and `p1-s6-fix-therefore` are applied, the new "Their depletion therefore drives..." plus the kept "is therefore critical..." puts two "therefore"s within ~30 words. Replacing one is recommended.

#### B.7 — Verdict line

End with one sentence on whether the paragraph is publication-ready after the staged edits, or whether another pass is needed.

> Para 1 is otherwise in strong shape. After these edits, the paragraph should be ready.

### Choosing the mode

| Signal | Mode |
|---|---|
| Full section / manuscript submitted | A (Clean output) |
| Single paragraph submitted | B (Professional pass) |
| User says "polish", "final pass", "native editor", "paragraph-by-paragraph", "approve each change" | B |
| User says "proofread", "copyedit the manuscript", "fix the English" with a multi-section text | A |
| Cover letter or abstract | B (paragraph-scale; benefits from per-sentence inventory) |
| Ambiguous | Run one Mode B sample on the first paragraph; ask which mode to continue with |

## Output rules

- **Flag structural gaps; do not fill them.** If the Discussion lacks a limitations paragraph, or the Introduction has no explicit gap statement, note it clearly — but do not write the missing content unless the author explicitly asks. Inventing scientific content is not proofreading.
- **Flag division-of-labor violations.** Point out interpretation that has leaked into a caption, or a Results paragraph that only lists facts with no trend statement.
- **Never alter scientific claims, numbers, or conclusions.** If a number looks internally inconsistent, raise it as a question — do not "fix" it.
- **Preserve the author's voice** where the English is already acceptable.

## What not to do

- Do not start line-editing before the structural pass — local fixes get wasted when structure changes.
- Do not leave fact/interpretation mixing unaddressed in captions or Results.
- Do not fill structural gaps the author did not ask to be written.
- Do not convert every passive blindly — apply the four justified-passive exceptions.
- Do not inflate the reason annotations; name the principle briefly and move on.

## Self-check before delivering output

Run the items relevant to the chosen mode before returning the output. Fix or flag any failure.

**Shared (both modes)**

- [ ] No scientific claim, number, or conclusion has been altered.
- [ ] Structural gaps are flagged, not silently filled with invented content.
- [ ] AMA number-style spot check passed: "hospital day" + Arabic, decimals under 1 have leading zero, percentages use the % sign, *P* is italicized.

**Mode A (Clean output)**

- [ ] Every track-change entry names a principle (deletion test, passive→active, justified passive #N, vague quantifier→specific value, topic/stress reorder, AMA number style, etc.).
- [ ] "Questions to the author" block exists and lists any unresolved ambiguity.

**Mode B (Professional pass)**

- [ ] Sentence inventory table has one row per sentence with `S`, `Words`, `Voice`, `Notes`.
- [ ] All five sections (A grammar / B word choice / C length / D logical flow / E positive notes) appear, or absence is justified by zero genuine issues.
- [ ] Every staged edit has an edit ID in `p{N}-s{M}-{slug}` format, a strikethrough+bold diff, and a 3–5 sentence reasoning.
- [ ] Interaction collisions between staged edits are stated explicitly.
- [ ] Verdict line closes the paragraph.
